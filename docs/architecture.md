# Architecture

## How it works

1. Registers as provider `glueclaw` with three models (opus, sonnet, haiku)
2. `createStreamFn` spawns `claude -p --output-format stream-json`
3. Scrubs detection triggers from the system prompt (see
   [detection-patterns.md](detection-patterns.md))
4. Translates NDJSON stream events into OpenClaw's AssistantMessageEventStream
5. Bridges OpenClaw gateway tools via MCP loopback
6. Persists sessions to `~/.glueclaw/sessions.json` for multi-turn resume
7. Translates response tokens back to gateway format

## Source files

| File                   | Purpose                                                                           |
| ---------------------- | --------------------------------------------------------------------------------- |
| `index.ts`             | Plugin registration, model catalog, stream function wiring                        |
| `src/stream.ts`        | Subprocess spawning, NDJSON parsing, scrub chain, session persistence, MCP bridge |
| `src/openclaw.d.ts`    | Type declarations for the host-provided OpenClaw plugin SDK                       |
| `install.sh`           | Installer: deps, plugin registration, model config, auth, gateway MCP patch       |
| `openclaw.plugin.json` | Plugin manifest                                                                   |
| `tsconfig.json`        | TypeScript compiler configuration                                                 |

## Scrub chain

In `src/stream.ts`, the scrub chain replaces detection triggers before passing
the system prompt to the Claude CLI:

```typescript
.replace(
  /personal assistant running inside OpenClaw/g,
  "personal assistant running inside GlueClaw",
)
.replace(/HEARTBEAT_OK/g, "GLUECLAW_ACK")
.replace(/reply_to_current/g, "reply_current")
.replace(/\[\[reply_to:/g, "[[reply:")
.replace(/openclaw\.inbound_meta/g, "glueclaw.inbound_meta")
.replace(/generated by OpenClaw/g, "generated by GlueClaw")
```

## Translation layer

Response tokens are translated back to gateway format in both streaming deltas
and `endStream`:

```typescript
text.replace(/GLUECLAW_ACK/g, "HEARTBEAT_OK");
text.replace(/reply_current/g, "reply_to_current");
text.replace(/\[\[reply:/g, "[[reply_to:");
```

## Session persistence

- Saved to `~/.glueclaw/sessions.json`
- Claude subprocess runs from `cwd: ~/.glueclaw`
- On resume: `--system-prompt` is skipped (session already has it)
- New sessions: scrubbed prompt passed via `--system-prompt`

## MCP bridge

The installer patches `server-*.js` in the OpenClaw dist to expose
`__GLUECLAW_MCP_PORT` and `__GLUECLAW_MCP_TOKEN` via `process.env`. The stream
function reads these, writes a temporary MCP config, and passes
`--strict-mcp-config --mcp-config <file>` to the Claude subprocess. This gives
it access to 19 OpenClaw gateway tools.
