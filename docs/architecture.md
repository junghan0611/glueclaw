# Architecture

GlueClaw is an OpenClaw provider plugin that routes inference through the real
`claude` CLI binary, using your Claude Max subscription instead of API keys.

## How it works

1. Registers as provider `glueclaw` with models: glueclaw-opus, glueclaw-sonnet,
   glueclaw-haiku
2. `createStreamFn` spawns `claude -p --output-format stream-json` as a
   subprocess
3. Scrubs Anthropic detection triggers from the system prompt (see
   [detection-patterns.md](detection-patterns.md))
4. Translates NDJSON stream events into OpenClaw's AssistantMessageEventStream
   format
5. OpenClaw gateway tools bridged via MCP loopback server
6. Sessions persist to `~/.glueclaw/sessions.json` and resume across gateway
   restarts via `--resume`
7. Response tokens (GLUECLAW_ACK, reply_current) translated back to
   gateway-expected format

## Source files

- `index.ts` - plugin registration, model catalog, stream function wiring
- `src/stream.ts` - core: subprocess spawning, NDJSON parsing, MCP bridge,
  session persistence, token translation
- `src/openclaw.d.ts` - type declarations for the host-provided OpenClaw plugin
  SDK
- `install.sh` - automated installer: deps, plugin registration, model config,
  auth, gateway MCP patch
- `openclaw.plugin.json` - plugin manifest
- `tsconfig.json` - TypeScript compiler configuration

## Scrub chain

The scrub chain in `src/stream.ts` removes Anthropic detection triggers from the
system prompt before passing it to the Claude CLI:

```typescript
.replace(/personal assistant running inside OpenClaw/g,
  "...GlueClaw")
.replace(/HEARTBEAT_OK/g, "GLUECLAW_ACK")
.replace(/reply_to_current/g, "reply_current")
.replace(/\[\[reply_to:/g, "[[reply:")
.replace(/openclaw\.inbound_meta/g,
  "glueclaw.inbound_meta")
.replace(/generated by OpenClaw/g,
  "generated by GlueClaw")
```

## Translation layer

Response tokens are translated back to the format the gateway expects. Applied
in both streaming deltas and endStream:

```typescript
text.replace(/GLUECLAW_ACK/g, "HEARTBEAT_OK");
text.replace(/reply_current/g, "reply_to_current");
text.replace(/\[\[reply:/g, "[[reply_to:");
```

## Session persistence

- Sessions saved to `~/.glueclaw/sessions.json` (survives gateway restarts)
- Claude subprocess runs from `cwd: ~/.glueclaw` so Claude Code stores session
  data there
- On resume, `--system-prompt` is NOT passed (session already has it)
- New sessions get the scrubbed prompt via `--system-prompt`

## MCP bridge

The installer patches the OpenClaw gateway's `server-*.js` to expose the MCP
loopback port and token via environment variables. The stream function then:

1. Reads `__GLUECLAW_MCP_PORT` and `__GLUECLAW_MCP_TOKEN` from `process.env`
2. Writes a temporary MCP config file
3. Passes `--strict-mcp-config --mcp-config <file>` to the Claude subprocess
4. This gives the Claude subprocess access to 19 OpenClaw gateway tools
   including message, cron, sessions, memory, browser, and web search
